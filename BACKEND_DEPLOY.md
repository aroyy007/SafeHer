# SafeHer — Backend Deployment Guide

Three free / cheap platforms, ranked for the CUET SciBlitz demo (July 17, 2026):

| Platform | Cost | RAM | Sleeps? | Best for |
|---|---|---|---|---|
| **Render** (free) | $0 | 512 MB | Yes (15 min) | Default; sleep is fine if you wire UptimeRobot |
| **Render** (Standard) | $7/mo | 2 GB | No | When you want the full Bengali SBERT model loaded |
| **Railway** | $5 trial credit, then ~$5/mo | 8 GB | No | Easiest "just works" with no sleep |
| **Hugging Face Space** | $0 (CPU basic) | 16 GB shared | No (always warm) | If your team already uses HF and wants free GPU/CPU |

The repo is already wired for **Render Blueprints** — `render.yaml` is committed at the root. Pick any of the three options below.

---

## Option A — Render (recommended default)

### A.1 Create the Blueprint service

1. Sign in at https://dashboard.render.com with GitHub.
2. Click **New + → Blueprint**.
3. Pick the **`aroyy007/SafeHer`** repo (Render asks for permission first time).
4. Render detects `render.yaml` and shows a preview of the `safeher-api` service.
5. Click **Apply**.
6. Render starts the first deploy. While it runs, scroll to the service → **Environment** tab and fill in the secrets listed below.

### A.3 Pick the right RAM tier

**Free tier (512 MB) — must use LITE_MODE:**
- `LITE_MODE=true` → skips Bengali SBERT + ChromaDB seeding on boot
- `ENABLE_BENGALI_EMBEDDER=false` → uses the 80 MB English MiniLM model
- The chat endpoint returns a friendly "lite mode" canned reply when the KB is unavailable
- /sos, /sos/alert, /incidents, /health still work normally

**Standard plan ($7/mo, 2 GB) — recommended for the actual demo:**
- `LITE_MODE=false`
- `ENABLE_BENGALI_EMBEDDER=true`
- Bengali SBERT loads (~440 MB) plus the rest of FastAPI fits comfortably

### A.4 Keep it awake

Render's free tier sleeps after 15 min of no traffic. The first request then takes ~30 s (cold start). For the demo:

1. Sign up at https://cron-job.org (free).
2. Create a job: `GET https://safeher-api.onrender.com/health` every 14 minutes.
3. The `/health` endpoint is intentionally tiny — no DB calls, no model loads — so it's safe to ping.

The `render.yaml` already sets `healthCheckPath: /health` so Render's own internal checks also keep it warm.

### A.5 Verify

```bash
curl https://safeher-api.onrender.com/health
```

Should return:
```json
{"status":"lite","lite_mode":true,"graph":{...},"knowledge_base":{...}}
```

or, on Standard:
```json
{"status":"ok","lite_mode":false,"graph":{"loaded":true,...},"knowledge_base":{"document_count":42}}
```

## Option B — Railway (no sleep, $5 trial credit)

Railway is the easiest "sleep-free" deploy if Render's 15-min idle gets annoying during rehearsals.

### B.1 One-click

1. Sign in at https://railway.app with GitHub.
2. **New Project → Deploy from GitHub → SafeHer**.
3. Railway auto-detects the repo. Click **Add variables** and paste the same env-var table from **A.2** above.
4. **Settings → Root Directory**: `backend`
5. **Settings → Build Command**: `pip install --upgrade pip && pip install --no-cache-dir -r requirements.txt`
6. **Settings → Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
7. Click **Deploy**. First deploy takes ~3-4 minutes (builds wheels).
8. **Settings → Generate Domain** → Railway gives you `https://safeher-api.up.railway.app`.

### B.2 Why Railway for this demo

- **No sleep.** Always warm, perfect for live demo.
- **8 GB RAM by default** → you can run the Bengali SBERT model on the free trial.
- $5 trial credit is enough for ~30 days of demo-tier usage.

### B.3 After the trial

Set a $5/mo spending cap (Railway → Settings → Usage Limits) to avoid surprise charges.

## Option C — Hugging Face Space (Docker SDK)

HF Spaces give you a generous CPU container for free, with no sleep. Useful if your team is already on HF.

### C.1 Caveats

- HF Spaces expect a `Dockerfile` or `app.py` at the repo root, but our repo has `backend/` as a subfolder. We work around this by mounting `backend/` via a custom Dockerfile.
- HF Spaces expect the app to listen on **port 7860** (Render/Railway use `$PORT`). The start command must override.
- HF Spaces are public by default; secrets are stored as Repo secrets.

### C.2 Files to add

Create `Dockerfile` at the **repo root**:

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# System deps for geopandas/shapely wheels
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

COPY backend/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY backend/ .

ENV PORT=7860
EXPOSE 7860

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "7860"]
```

Create `README.md` for the Space (HF needs it) — but **don't replace the existing one**. Instead, name it `README_HF.md` and tell HF to use it via the Space UI.

### C.3 Create the Space

1. Sign in at https://huggingface.co.
2. **New Space → Docker → CPU basic (free) → name: `safeher-api`**.
3. **Files → upload the Dockerfile** you just created, **upload the entire `backend/` folder contents** at the repo root (flatten the structure: `main.py`, `requirements.txt`, `core/`, `services/`, etc., all at the root of the Space repo).
4. **Settings → Variables and secrets** → paste the env-var table from **A.2**.
5. Click **Build**. First build takes ~5 min.

### C.4 Verify

```bash
curl https://aroyy007-safeher-api.hf.space/health
```

Returns the same `/health` JSON. Public URL pattern is `https://<username>-<space-name>.hf.space`.

### C.5 When to use this option

- Your team already has an HF org and quota.
- You want free, no-sleep, no credit-card infra.
- You're comfortable flattening the backend folder structure for the deploy.

For most teams, **Railway is the lowest-friction "sleep-free" option**. Pick Render if you want the demo to live on a permanent URL without thinking about trial credits.

## Which option should I pick?

```
Want zero-config and don't mind sleep?   → Render free + UptimeRobot
Want zero sleep + simplest?             → Railway
Want free + on HF already?              → HF Space
Want full Bengali SBERT model live?     → Render Standard ($7/mo)
```

For the CUET SciBlitz demo (Jul 17, 2026), the recommended setup is:

1. **Deploy frontend to Netlify** (see [DEPLOYMENT.md](DEPLOYMENT.md))
2. **Deploy backend to Render Standard ($7/mo)** so the Bengali SBERT model loads
3. **Set LITE_MODE=false** so chat works in full Bengali
4. **Add cron-job.org keep-alive** anyway (defense in depth)
5. **Wire SMTP** via Gmail app password or SendGrid so `/sos/alert` works as a backup to EmailJS

If you can't justify $7/mo, use Render free + `LITE_MODE=true`. The chat endpoint will gracefully tell the user "lite mode active" and the rest of the demo (SOS, routing, heatmap, contacts) still works.
