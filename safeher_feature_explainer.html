SafeHer feature-by-feature explainer for teammates

SOS emergency button

The most important feature — must always be one tap away

Imagine a woman walking home at night feeling unsafe. She opens SafeHer and sees one big red button. She presses and holds it for 3 seconds. That's it. The app does everything else automatically.

  • 1User holds the big red SOS button for 3 seconds (prevents accidental presses)
  • 2Phone vibrates to confirm — screen turns full red
  • 3Her GPS location is captured instantly via the browser Geolocation API
  • 4An emergency email fires to ALL trusted contacts: "Nadia needs help. Live location: [link]. Time: 10:34 PM"
  • 5Her location link goes live on Firebase — contacts can track her in real time
  • 6Screen collapses to ONLY: contact list (who was notified), cancel button, emergency numbers (999, 10921)
  • 7Nothing else — no menus, no navigation, no distractions
Geolocation API Firebase RTDB EmailJS Vibration API 100% free
Who builds thisFrontend dev — React state machine for SOS active/inactive modes
Key rule3-second hold. No confirm dialog. Never more than 1 tap to reach the button

Voice-activated SOS

Hands-free emergency — the feature no existing app has in Bengali

Imagine the user's phone is in her bag. She can't safely take it out. She quietly says "বাঁচাও" (save me) — and SafeHer activates emergency mode automatically. No touching the phone required.

  • 1When the user opens SafeHer, the microphone starts listening silently in the background
  • 2The browser's built-in speech engine (Web Speech API) converts speech to text continuously
  • 3Our code checks if the transcript contains any trigger phrase: "help", "bachao", "বাঁচাও", "emergency", "amake bachao"
  • 4If a trigger is detected — same result as pressing the SOS button
  • 5If the mic disconnects (tab switch, browser pause) — it auto-restarts after 1.5 seconds
Web Speech API bn-BD locale Zero ML needed No cost
Who builds thisFrontend dev — ~40 lines of JavaScript, runs on page load
Demo momentDuring presentation: say "বাঁচাও" out loud. SOS fires. No judge will forget this.

  • 3Her GPS position updates to Firebase every 5 seconds
  • 4Anyone with the link opens it and sees her live dot on a map

Safe route recommendation

The AI X-factor — built on real Bangladesh crime data

Google Maps shows you the fastest route. SafeHer shows you the safest one. It avoids dark streets, areas with past harassment reports, and unlit alleys — even if that means walking 3 minutes longer.

  • 1User types: "From GEC Circle to Chittagong Medical College"
  • 2Our backend has pre-loaded the entire Chittagong walking map (from OpenStreetMap)
  • 3Every road segment has a safety score based on: street lighting (OSM data), past incidents (CrimeDataBD — 6,574 real Bangladesh crime records), road type (footways are safer than trunk roads)
  • 4Our modified A* algorithm finds the route with the highest total safety score
  • 5Map shows TWO routes: safe route in teal (thicker), fast route in amber (thinner)
  • 6User can toggle between them and see why one is safer
AI / ML — core feature FastAPI (Python) NetworkX A* osmnx CrimeDataBD
Who builds thisBackend dev — Python script pre-computes graph, FastAPI serves routes
Critical rulePre-compute the graph LOCALLY. Never download the map during a live demo — it takes 4 minutes and will crash.

Community safety map

Women report unsafe spots — the map learns and warns others

Imagine a neighborhood where 5 women have reported harassment near a particular bus stop after 8 PM. SafeHer shows that area as red on the map — so the next woman traveling that route is warned before she gets there.

  • 1User taps any point on the map to drop a pin
  • 2A form pops up: category (eve teasing / stalking / unsafe lighting / assault / other), description, time of day
  • 3Her report saves to Supabase database with her GPS coordinates
  • 4Map immediately shows colored dots: red = assault/rape area, amber = harassment zone, green = reported safe
  • 5When many reports cluster in one area, Mapbox shows a heatmap bubble (darker = more dangerous)
  • 6These reports ALSO feed into the safe route algorithm — reported areas raise safety_cost for nearby roads

AI safety assistant (RAG chatbot)

Answers emergency questions — never makes up phone numbers

A user types "I'm being followed, what do I do?" and gets a calm, specific 3-sentence answer in Bengali or English. It only answers from our verified database — it cannot hallucinate a wrong emergency number.

  • 1User types a question in the chat (Bengali or English)
  • 2Our system searches our ChromaDB database of 30+ verified facts (emergency numbers, laws, guidance)
  • 3The 3 most relevant facts are retrieved and sent to Groq (Llama 3.1 70B AI model)
  • 4The AI is strictly told: "Only answer from these facts. If you don't know, say call 999."
  • 5Response appears in under 1 second — maximum 3 sentences
  • 6If user types in Bengali — it responds in Bengali
Example: User types "কেউ আমাকে অনুসরণ করছে" (someone is following me)
Bot responds: "এখনই ৯৯৯ কল করুন। কাছের মসজিদ, ফার্মেসি বা ব্যস্ত দোকানে ঢুকুন। বাড়ির দিকে যাবেন না।"
RAG pipeline Groq Llama 3.1 70B ChromaDB Free tier

Trusted circle

Her private safety network — parents, siblings, close friends

Before using SafeHer, the user adds 3-5 trusted people. When anything bad happens, these people get notified instantly. They can see her location live without having to call her and potentially put her in danger.

  • 1User goes to Settings → adds contacts: name + email (or phone for future SMS)
  • 2Contacts are stored in Supabase under her account
  • 3When SOS fires — ALL contacts get email automatically via EmailJS
  • 4Email contains: her name, timestamp, a live tracking link, emergency numbers
  • 5On the SOS active screen — she sees all contacts listed with a green "NOTIFIED" badge
  • 6Contacts can open the tracking link on any browser — no app needed

Key UX ruleContacts don't need to install anything. Link opens in any browser.

Disguise mode

Hides SafeHer — looks like a calculator to anyone watching

Imagine a woman is in a threatening situation where the person threatening her can see her phone screen. She can't be seen using a "safety app." She triple-taps the logo — the screen instantly looks like a calculator. But underneath, everything is still running: SOS listening, voice trigger active, location sharing on.

  • 1User triple-taps the SafeHer logo (3 quick taps)
  • 2The entire UI swaps to a calculator skin — buttons, display, the works
  • 3Behind the scenes: SOS hold-to-activate still works (hold the "=" button for 3 seconds), voice trigger still listens, Firebase location still updating
  • 4No sounds play in disguise mode — completely silent
  • 5To exit: type a secret code (e.g. "1234=") on the calculator
This is the feature that makes SafeHer genuinely different from every other safety app. Women in focus groups consistently ask for this. Judges will remember it.
