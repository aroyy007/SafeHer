CrimeDataBD — Primary dataset

Core · must use
6,574 crime records 36 features per record CSV format CC BY 4.0 Free download

Bangladesh crime records covering Murder, Rape, Assault, Robbery, Kidnap. Includes lat/lng coordinates, victim gender, time of day, weather, population density, literacy rate, poverty rate. Built from newspaper archives + National Census + Weather API. The only ML-ready Bangladesh crime dataset with geographic coordinates.

How you use itFilter rows where victim_gender == "female". Extract lat/lng. Fit a KDE model on those coordinates. Use the KDE density surface to score every OSM road segment — higher crime density nearby = lower safety score on that edge.
data.mendeley.com/datasets/d7vzpcdzcw/4

OpenStreetMap via osmnx

Core · must use
Full walk network Python library ODbL license Free API

The entire walkable road network of Chittagong including road type (highway tag), street lighting (lit=yes/no), surface type, and road names. Accessed via the osmnx Python library which calls the Overpass API and returns a NetworkX graph directly usable for A* pathfinding.

How you use it
1Run ox.graph_from_place("Chittagong, Bangladesh", network_type="walk") — once, locally.
2Read data.get("lit") on each edge for lighting score.
3Read data.get("highway") for road type score.
4Save as chittagong_walk.graphml. Commit to repo. Never re-download at runtime.
github.com/gboeing/osmnx · pip install osmnx

Geofabrik Bangladesh OSM extract

Alternative · offline use
PBF format Full country extract Updated daily Free download

Full Bangladesh OSM data as a PBF file, updated daily. Use this if osmnx's Overpass API calls are slow or rate-limited in your area. Can be read with pyrosm instead of osmnx for offline processing of large areas.

How you use itOnly needed if you want to process the entire Bangladesh road network offline, or if Overpass API times out for Chittagong. For the hackathon, osmnx is simpler.

Used in: chatbot generation + RAG embeddings

Groq — llama-3.3-70b-versatile

Core · chatbot LLM
Free tier ~200–400 tok/s Bengali capable No training needed

The active replacement for Llama 3.1-70b on Groq (3.1 was deprecated December 2024). Sub-second inference on the free tier. Strong Bengali comprehension at 70B scale. You call this via API — no download, no GPU, no hosting cost.

How you use itPOST to api.groq.com/openai/v1/chat/completions with model: "llama-3.3-70b-versatile", temperature: 0.05, max_tokens: 150. System prompt contains retrieved RAG passages. User message is the original query.
console.groq.com — free API key

l3cube-pune/bengali-sentence-similarity-sbert

Core · RAG embeddings
~80MB RAM Sentence-level embeddings Bengali + English Free · HuggingFace

Empirically best model for Bengali RAG retrieval (verified by Bangla-RAG/PoRAG project). SBERT-based so it produces sentence-level vectors directly — no pooling step needed. Handles code-mixed Bengali-English queries better than BanglaBERT. Fits on Render's 512MB free tier.

How you use itSentenceTransformer("l3cube-pune/bengali-sentence-similarity-sbert"). Encode your knowledge base chunks once at startup. Encode every user query at request time. ChromaDB finds the closest chunks by cosine similarity.
huggingface.co/l3cube-pune/bengali-sentence-similarity-sbert

csebuetnlp/banglabert (BUET NLP)

Reference · normalization only
ELECTRA-based 27.5GB pretraining data NAACL 2022 Free · HuggingFace

You are not using BanglaBERT as an embedding model (l3cube is better for RAG). You ARE using the csebuetnlp/normalizer package that ships alongside it — a mandatory text normalization pipeline for Bengali Unicode cleaning, hasanta fixes, and punctuation normalization.

How you use itpip install git+https://github.com/csebuetnlp/normalizer then from normalizer import normalize. Call normalize(text) on every Bengali string before embedding. Never embed raw Bengali without this step.
github.com/csebuetnlp/banglabert

Used in: safety score weights + judging justification

SafetiPin 9-parameter safety audit framework

Academic reference · cite this
Published: Gender & Development 2015 9 safety parameters Used in 50+ cities Free to read

The academic standard for measuring urban safety for women in South Asian cities. Nine parameters: lighting, openness, visibility, people presence, security, walk path, transportation, gender usage, incident history. Your route safety score weights (0.35 lighting, 0.40 incident, 0.25 road type) are derived from this rubric — cite it in your Model and Data Card.

How you use itYou do not download data from SafetiPin. You cite their published methodology to justify your safety score formula. When a judge asks "why these weights?" — "based on the SafetiPin safety audit framework published in Gender and Development 2015" is a complete, credible answer.
doi.org/10.1080/13552074.2015.1013669 — Viswanath & Basu 2015

The project that empirically tested sagorsarker/bangla-bert-base, csebuetnlp/banglabert, and l3cube-pune/bengali-sentence-similarity-sbert for Bengali RAG retrieval quality and found the l3cube model most effective. Your embedding model choice is based on this work — cite it.

How you use itStudy their retrieval pipeline and ChromaDB integration. Their exact requirements (chromadb==0.5.0, sentence-transformers==3.0.1) are a safe starting point for your own requirements.txt.

Normalizes Bengali Unicode — fixes zero-width joiners, hasanta characters, punctuation variants, and numeral inconsistencies. Required before any Bengali text reaches an embedding model. Ships as a standalone pip package separate from the BanglaBERT model weights.

shuaib128/Banglish (phonetic rules)

Reference · transliteration rules
JavaScript Avro phonetic rules MIT license Free

Chrome extension implementing Avro-style phonetic Banglish-to-Bengali transliteration. The phonetic mapping rules (which romanized sequences map to which Bengali script characters) are the reference for the transliteration table in your backend preprocessor.

How you use itRead the Phonetic class in this repo to understand the mapping rules. Port those rules to your Python transliterator. You do not import this library — it is JavaScript. You replicate the phonetic logic in Python.
